## Design Context

### Users
Rune 프레임워크를 사용하는 프론트엔드 개발자. 개발 중 컴포넌트 트리, 상태, 이벤트, 성능을 디버깅하는 맥락에서 사용. 빠르게 문제를 파악하고 해결하는 것이 목적.

### Brand Personality
실용적, 정밀한, 도구다운 — 전형적인 개발자 도구의 느낌. 꾸밈보다 기능, 장식보다 정보 밀도.

### Aesthetic Direction
- **참고**: Astro Dev Toolbar — 깔끔하면서 예쁜 DevTools
- **반참고**: 지나치게 화려하거나 AI가 만든 느낌의 제네릭한 UI
- **테마**: 다크 모드 전용 (DevTools 표준)
- **액센트 컬러**: Rune 브랜드 블루 `#1c75ff` 기반
- **타이포그래피**: 코드/데이터는 모노스페이스, UI 라벨은 시스템 폰트
- **스페이싱**: Rune 프로젝트의 4px 기반 그리드 유지

### Design Principles
1. **정보 밀도 우선** — 공간을 낭비하지 않는다. 개발자는 한눈에 많은 정보를 봐야 한다.
2. **기능이 장식을 이긴다** — 그라디언트, 그림자, 애니메이션은 정보 전달에 도움이 될 때만 사용.
3. **컬러는 의미를 담는다** — 색상은 상태(성공/경고/에러)와 카테고리 구분 용도로만. 장식적 컬러 사용 금지.
4. **일관된 패턴** — 같은 유형의 정보는 항상 같은 방식으로 표현. 패널마다 다른 규칙을 만들지 않는다.
5. **Rune 브랜드와 연결** — #1c75ff 블루를 액센트로, DevTools가 Rune 에코시스템의 일부임을 자연스럽게 전달.
